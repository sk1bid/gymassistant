# GYM.assistant Mini App

Веб-интерфейс бота. Весь пользовательский функционал — тренировка, конструктор
программ, каталог упражнений, история и графики — живёт здесь. Бот остался точкой
входа и, что важнее, единственным, кто умеет уведомлять.

## Почему гибрид, а не «просто веб»

Mini App не может напоминать об окончании отдыха. Закрытая страница не существует,
JS-таймер вместе с ней останавливается, Web Push внутри телеграмовского вебвью нет.
А вся ценность продукта в том, что телефон лежит на скамье экраном вниз и пингует,
когда отдых кончился.

Поэтому таймер отдыха — это строка в таблице `rest_timer`, а не таймер в браузере:

* **Mini App** ставит таймер (просто запись в БД) и, пока открыт, рисует по нему
  живой отсчёт;
* **бот** (`workers/rest_notifier.py`) раз в пять секунд разбирает активные таймеры
  и шлёт пинги в чат;
* кнопка «Закончить отдых» в любом из двух гасит одну и ту же строку.

Из этого следует правило, которое нельзя нарушать: **каждый пинг — новое сообщение
с удалением предыдущего, а не `editMessageText`.** Редактирование сообщения в Telegram
не даёт ни пуша, ни вибрации. Заменить это на edit — значит убрать уведомления,
то есть весь смысл.

## Что переиспользовано

`database/orm_query.py` (~60 функций) и `services/neiro_api.py` импортируются **без
единого изменения** — они ничего не знают про Telegram, только про SQLAlchemy.
Handlers и клавиатуры бота (≈5000 строк сборки меню) фронт заменяет целиком.

Дописано только то, чего в боте не было:

| Модуль | Зачем |
|---|---|
| `database/orm_extra.py` | таймер отдыха; история по *личности* упражнения, а не по строке `Exercise` |
| `services/workout.py` | текущий шаг тренировки как чистая функция от плана и факта |
| `services/predictor.py` | LSTM для любого упражнения, а не только для жима по названию |

## Блокировка Telegram обходит стороной

Провайдер режет подсети Telegram в обе стороны, поэтому бот ходит наружу только через
SOCKS5 и работает поллингом. **Mini App это не касается:** страницу телефон грузит
напрямую с `sk1bid.ru`, а подпись `initData` проверяется локально (HMAC-SHA256, ключ
`"WebAppData"`). Обращений к `api.telegram.org` у этого сервиса нет вообще — и прокси
ему не нужен.

## Архитектура

```
miniapp/
  config.py       настройки из окружения
  auth.py         проверка подписи initData — единственное место, знающее про Telegram
  db.py           сессия БД на запрос
  deps.py         пользователь текущего запроса (заводится при первом входе)
  ownership.py    «это точно твоё?» — id приезжают с фронта, им верить нельзя
  schemas.py      что приходит
  serializers.py  что уходит
  state.py        сборка состояния тренировки
  routers/        HTTP, по модулю на раздел
  main.py         сборка приложения
  static/         фронт: ES-модули без сборки, по экрану на файл
```

Состояние тренировки **не хранится между запросами**: план — это упражнения дня,
факт — записанные подходы, а текущий шаг вычисляется как первое расхождение между
ними. Поэтому закрытое приложение, перезапуск пода и второй телефон ничего не ломают.
В боте это состояние жило в FSM (`MemoryStorage`), и рестарт обрывал тренировку.

## Запуск

Локально (из каталога `gymassistant/`):

```bash
DB_URL=sqlite+aiosqlite:///./demo.db \
MINIAPP_BOT_TOKEN=<токен_тестового_бота> \
  ./miniapp/.venv/bin/uvicorn miniapp.main:app --port 8099
```

Тесты:

```bash
./.venv-bot/bin/pytest tests/ -q
```

Снаружи — `https://sk1bid.ru/gym/` (nginx проксирует на `127.0.0.1:8099`),
в Telegram — кнопка меню бота.

### Отладка фронта в обычном браузере

Вне Telegram `initData` взять неоткуда, поэтому `tg.js` читает его из
`localStorage.devInitData`. Дырой это не является: подпись всё равно проверяет сервер,
а подписать `initData` можно только токеном бота — без валидной подписи будет 401.

## Проверка безопасности

Подпись обязана отвергать подделку — без этого любой пишет подходы в чужой аккаунт.
Это закреплено тестом `test_forged_init_data_rejected`: в валидном `initData`
подменяется `user.id`, ответ — 401. Плюс `test_cannot_touch_other_users_data`:
чужие программы и дни не читаются и не правятся по id.
